Hugo ile Statik Bir Websitesi Nasıl Yapılır? - Beşinci Bölüm : Sitemizi Github'da Yayınlayalım

1/Sep 2016
HUGO GIT WEBSİTE GITHUB HOSTING
Like Google +1 Tweet

Nihayet bugün, Hugo ile statik bir websitesi nasıl yapılır serimizi beşinci bölümü ile sonlandırıyoruz.

Birinci bölümde Hugo kurulumunu, ikinci bölümde Hugo ile Siteyi Lokal Bilgisayarda Oluşturma‘yı, üçüncü bölümde Temamızı Özelleştirme‘yi, dördüncü bölümde ise post sayfalarını editleme konusunu işlemiştik.

Bugünkü konumuz, sitemizi ücretsiz olarak Github‘da barındırma ve bu işlemi git vasıtası ile nasıl yapacağımız.

Github Hakkında

Github yazılım geliştiricilerin kullandığı Dünya’nın en önemli ortak - paylaşılan sitelerinden biri (hub). Bu site 5 milyondan fazla açık kaynak kodlu yazılıma ev sahipliği yapıyor.

Özellikle açık kaynak kodlu (open source) yazılımlar Github‘a yükleniyor ve Dünyanın her tarafından programcılar bu yazılımları her an geliştirme ve destekleme şansı elde ediyorlar.

Siz de herhangi bir projeye destek vermek isterseniz yapmanız gereken yazılım projesini bilgisayarınıza indirmek (fork-clone) ve yazılım üzerinde yaptığınız değişiklikleri tekrar Github’a yüklemek. (pull request)

Yaptığınız değişikliklerin projeye eklenebilmesi için projeyi sürdürenlerin onaylaması gerekiyor.

Github’ı Ücretsiz Hosting Servisi Olarak Kullanmak

Github aynı zamanda (Jekyll, Hugo) v.s. S.S.G.ler ile yapılan statik web sitelerine ücretsiz barındırma servisi (hosting) sağlaması ile göz dolduruyor.

Bu sayede kendi blogunuzu teorik olarak 0 masrafla hayata geçirmeniz mümkün. (Hosting masrafı dışında bir websitenin tek masrafı -eğer alacaksanız- custom domain ücretidir.)

Sitenize özel bir domain ismi almak isterseniz .com ve .net gibi çok kullanılanlar dışındaki domainleri bedavaya getirebilirsiniz.

Github görüp görebileceğiniz en sağlam (stabil) hosting servislerinden, site yüzbinlerce geliştiriciye hizmet verdiğinden 365 gün full hizmete açık. Down (hizmet dışı kalma) istisna süresi 0.0001% seviyesinde.

Github hakkında daha önce oldukça bilgi vermiştim zaten. Lütfen o makaleme bir göz atın.

Github’a Giriş

Github’ı kullanmak için üye olmak gerekiyor. Github ana sayfasında üye olun (Sign Up) ve kendinize bir kullanıcı ismi belirleyin.

Kullanıcı isminiz ile giriş yaptığınızda ana sayfada Start a project düğmesine basın. Projenize kullanıcıisminiz.github.io ismini verin ve Create repository yeşil düğmesine basın. (↑)

Web Site Dosyalarımızı Hazırlayalım

Sitemizi yapıp bitirdik, son olarak sitemizin ana klasöründe hugo komutunu verdik ve Github‘a yükleyeceğimiz dosyalar site ana klasörünün altında yeni oluşturulan public klasörümüzün altında hazır. (Foto 1 ve 2)


“Foto1: Hugo Websiteyi Oluşturdu - Generate” — “Foto2: Hugo Public Klasörü Görüntüsü”

ÖNEMLİ NOT: Siteyi generate etmeden (yani hugo komutunu vermeden önce) konfigürasyon dosyanızdaki (config.toml) baseURL kısmının karşısında “https://kullanıcıadınız.github.io" yazdığınızdan emin olun.

Yukarıdaki işlemi yapmadan siteyi oluşturduysanız public klasörünü silin, bu işlemi yapın ve sonrasında siteyi tekrar oluşturun(hugo). Aksi halde Github’da tema şablonlarını doğru olarak göremeyebilirsiniz.

Web Sitemize ‘de hadi “git” gari’ Diyoruz

1. Şimdi *public* klasörüne Powershell ile gidin. (D:\siteler\deneme\public)
2. Github'a yüklemek için bunu bir proje olarak tanıtalım. `git init` komutunu verin.
3. Klasör içindeki her şeyi komit için eklemek üzere işaretleyin. `git add .`
4. Dosyaları komitle. `git commit -m "ilk komitimiz"
5. Şimdi uzak sunucuyu tanıtalım. `git remote add origin https://github.com/kullaniciadiniz/kullaniciadiniz.github.io.git`
6. Evet, şimdi gidiyoruz: `git push -u origin master`

Github bize kullanıcı adımızı ve parolamızı soracak. Onlara cevap verin ve en geç 3 saniye içinde sitemiz yüklenmiş olacak. 1-2 saniye de sitenin kendine gelmesini bekleyin. Sitenizi tarayıcınızda açtığınızda artık sitenizin webde olduğunu göreceksiniz. Vauv!

Sitenize Custom (Özel) Domain İsmi Verme

kullanıcıisminiz.github.io ismi herkes için uygun olmayabilir. Siz daha profesyonel görünme adına ornek.net / www.ornek.com gibi bir isim kullanmak isteyebilirsiniz.

Github özel domain isimleri için destek sağlıyor. İsminizi aldıktan sonra yapmanız gereken iki (2) şey var:

  1. Domaini aldığınız şirketin control panelinde bir CNAME kaydı (duruma göre A record) oluşturup buraya Github’ın IP adreslerini vermek.
  2. CNAME isimli bir metin dosyası oluşturup bu dosyanın içine domain isminiz ne ise onu yazarak (örnek: www.domainisminiz.com) dosyayı sitenin static klasörünün içine atmak.

Detaylı bilgiye Github destek sayfalarından ulaşabilirsiniz.

Şükür, Hugo ile statik bir websitesi nasıl yapılır serimiz burada bitti. Bitmeyeceğini sanmıştım bir an.

Umarım faydalı olmuştur. Seride daha detaylı açıklamaya ihtiyaç duyduğunuz konuları sormaktan çekinmeyin lütfen. İyi çalışmalar…
