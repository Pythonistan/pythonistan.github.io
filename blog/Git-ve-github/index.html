Git Nedir? Nerede ve Nasıl Kullanılır?

20/Aug 2016
git github website
Like Google +1 Tweet

Git ile Tanışma

Bir önceki yazımda bahsettiğim Hugo ve Jekyll ile web sitesi yapma olayına girişmeden önce üzerinde durmamız gereken bir konu var: Git.

Bazı temel git komutlarını öğrenmemiz gerekiyor zira web sitemizi oluştururken dosya indirme ve siteyi oluşturduktan sonra ise sitemizi web server’a yüklemek için bu komutları kullanacağız.

Git bizim için sadece dosyalarımızı yedeklemek yada güncellemeleri tutmakla kalmayıp dosyaları web’e transfer etmede kullanacağımız bir yol olacak. Eskiden bu iş için ftp protokolu ve ftp programları kullanılırdı.

Git : Versiyon Kontrol Sistemi

Git yazılımlar için kullanılan bir çeşit versiyon kontrol sistemidir. Yazılımcılar bu tür sistemlerle kaynak kodunda zaman içinde yapılan değişimleri görüp yönetebilirler.

Versiyon kontrol sistemleri bir yazılımda herhangi bir anda yada herhangi bir kişi tarafından yapılan her türlü değişikliğin hepsinin kaydını tutar. Herhangi bir hata durumunda kolaylıkla eski stabil duruma dönüş olanağı verir.

Bu şekilde çok değerli olan kaynak kodumuza gelecek zararları önleneyebiliriz.

Git’in mucidi Linux’un da ilk geliştiricisi olan Linus Torvalds.

Git : Bizim İçin Anlamı

Yukarıda Git‘i tanımlayan ifadeler fazla soyut kalmış olabilir. Yada kendinize “Benim büyük yazılım projeleri ile ilgim yok, o halde neden git‘e ihtiyaç duyayım?” sorusunu soruyor olabilirsiniz.

Git, üzerinde yüzlerce kişinin aynı anda çalıştığı ve katkı verdiği büyük yazılım projeleri (Linux, v.d.) yada milyarlık firmaların üzerinde çalıştığı envai çeşit projeler (mesela kitap revizyonu, v.d.) için değil, her tür boyuttaki kişisel projelerde kullanabilecek imkanlar sağlıyor.

Bu sistemi anlamak adına basit bir örnek verelim: Üzerinde çalıştığımız bir word dosyamız olsun. Mesela uzunca bir doküman yazıyoruz. Dosyamızı her gün açıp bir şeyler ekleyip, çıkardığımızı varsayalım.

Böyle bir çalışmayı şu ana kadar yaptıysanız bazen yapmamanız gereken bir hata yaptığınızı, eklememeniz gereken bir şeyi eklediğinizi bazen de tersini yaptığınızı görmüşsünüzdür. Dolayısıyla kaçırdığınız bir şeyler hep olur.

Bazen de ana dosyanın üzerine kaydetmeyip farklı kaydet seçeneğini kullanarak birbirinin benzeri onlarca dosya oluşturduğunuzu şaşkınlıkla görür, hangisini en son oluşturduğunuzu anlamak için bir sürü zahmet çekersiniz.

Bazen bir dosyada bulunan bir bilgi diğerinde bulunmaz, o dosyadan diğer dosyaya taşımanız gerekir. Kısaca büyük bir zahmettir.

Git : Master ve Branch

Git bu tür durumlar için dosyanın ana kopyası (master) ve yukarıdaki örnekteki gibi kaydedilen diğer versiyonlar için branş kopyası (branch) konseptini sunuyor. Bir ana dal ve yüzlerce katkı verenin oluşturduğu yüzlerce dallardan oluşan bir ağaç gibi düşünün.

Branşda oluşan değişikliklerden memnunsanız branşı ana dala bağlayabilirsiniz. (merge)

Memnun değilseniz iptal edip ana kopyadan yeni bir dal oluşturabilir, kullanmadığınız dalları iptal edebilirsiniz.

Dosyalarımız git sisteminde otomatik olarak yedeklenir. Herhangi bir anda dosya üzerinde o ana kadar yapılan düzeltmeleri (commit) görebilir (git status) ve herhangi bir andaki duruma (statüye) dönebilirsiniz. (git reset --hard statü.nosu)

Git : Repository (Depo)

Git sisteminde oluşturduğumuz her bağımsız proje birimine repository deniyor. Repo’nun içeriği dosya, klasör, foto, video, excel tablosu, ve her tür veri seti olabilir.

Herhangi bir repo oluşturduğumuzda onun master (ana) kopyası hemen oluşturulur. Üzerinde çalışabileceğimiz branş kopyalarını biz oluşturacağız. Branş kopyaları üzerinde denediğimiz ve memnun kaldığımız değişiklikleri ana kopyayla birleştiriyoruz. (merge)

Git : Bizi Asıl İlgilendiren Komutlar

Git sistemi gerçekten çook geniş ve başlı başına uzmanlaşılabilecek bir alan. Biz asıl olarak git ile bir web server’a ulaşmak ve oradan birşeyler indirmek (pull - clone), Git sistemini kullanan en önemli sitelerden biri olan Github’a dosya yüklemek (push), repo oluşturup (init), repoya yapacağımız son değişiklikleri güncelleyip düzeltmek (commit) gibi işlemler yapacağız.

Hadi başlayalım.

Git kuruyoruz.

Git kurmak için Git websitesine gidip sisteminize uygun olan versiyonu indirin ve kurun.

Önemli: Kurulum sırasında ortam değişkenine (PATH) ekleme seçeneğini ikinci seçenek olarak seçin. Yani “Use Git from the Windows Command Prompt”. (hemen altta resim 1) Bu şekilde (Powershell)‘den sıkıntısız olarak git komutlarını çalıştıracağız. Diğer seçenekleri varsayılanlar olarak bırakın.

Git’i komut satırından çalıştıracağımız için daha önceki yazılardan aşina olduğunuz (Powershell)‘i yine çalıştırın. Kurulumun doğru olup olmadığını anlamak için herhangi bir klasör içinde git yazıp gelen cevaba bakabilirsiniz. (hemen altta resim 2)

Bir Repo Oluşturalım

Şimdi bilgisayarınızın içinde herhangi bir klasöre gidin ve şu komutu verin: git init. Klasörünüzün içinde bir kaç dosya bulunsun. Bu komut git’e bu klasörün içinde bir repo oluşturulacağını ve git sisteminin bu repoyu takibe başlayacağını söylüyor.

Daha sonra git add . komutunu vererek klasörümüzün içinde bulunan her şeyin (her tür dosya) repoya eklenmesini istiyoruz.

Şimdi git commit -m "dosyaları repoya ilk kez bagliyoruz" yazıp Enter’a basarak repoya eklediğimiz dosyaları tam o andaki haliyle ve durumuyla (son düzeltme durumu-last commit) repomuza bağlanması işlemini gerçekleştiriyoruz.


“Git ile Yukarıdaki Komutları Deneyelim”

Sıra Uzak Sunucu Olan Github’a Geldi

Lokalda (yerel bilgisayar) yapacağımız işlemler bitti. Şimdi sıra ekleyip repoya bağladığımız dosyalarımızı git vasıtasıyla Github‘a (yada Gitlab/Bitbucket, hangisini kullanıyorsanız ona) yüklemek.

İlk olarak Github sitesine gidin ve kayıt olun. (Ana ekranda sağ üstteki yeşil Sign up düğmesi) Kendinize kısa bir kullanıcı ismi belirleyin.

Kayıttan sonra kayıt sırasında verdiğiniz mail adresinize bir teyit maili gelecektir, onu teyit edin.

Kayıt işlemi tamamlandıysa şimdi ana ekranda Start a project düğmesine tıklayın. Reponuza test adını verin ve Create Repository düğmesine tıklayarak reponuzu oluşturun. (üstteki büyük resim)

Github’a Yükleme Yapalım

Şimdi Lokalde oluşturduğumuz repomuzu web’e yükleyelim. Komut satırı Powershell’de vereceğimiz komut: git remote add origin https://github.com/kullaniciadiniz/test.git (bu komutu git init komutunu verdiğiniz aynı klasörün içinden çalıştıracaksınız)

Son olarak aşağıdaki komutu girin: git push -u origin master

Github size şimdi kullanıcı adınızı ve parolanızı soracak. Kullanıcı adınızı girin ve Enter’a basın. Şifreniz istendiğinde yine aynı şekilde girin (şifreniz ekranda görülmeyecektir) ve Enter’a basın. Dosyalarınız webdeki alanınıza yüklenecek.


“Git ile Github’a Dosyaları Yüklüyoruz”

Bir - iki saniye sonra Github websitesini kontrol edin. Yüklediğiniz dosyaların test isimli reponuzun altında olduğunu göreceksiniz.

Şu andan itibaren takip edilen klasörünüzün içindeki dosyalarda yaptığınız her değişikliğin kaydı tutulacak ve commit (düzeltme) yaptığınız anda bu değişiklikler repo ile güncellenmek amacıyla tutulacaktır.

Tersi de geçerli. Yani uzak sürücüdeki (web) dosyalarda herhangi bir değişiklik düzeltmesi yaptıysanız bu düzeltmeler lokaldeki kopyasıyla eşitlenecektir. (Bunun için verilen komut: git pull origin master)

Komut Satırı Sistemlerinin Yaşattığı Duygular

Bizim gibi acemiler için komut satırı kendisinden saygı ile karışık korku beslenen, anlaşılmaz ve hatta biraz mistik bir konu gibi.

Her an bir yanlış yapıp sistemi mahvedeceğim duygusu komut satırını kullanırken çok kere aklıma gelmiştir.

İşletim sistemi olarak Windows ve Mac kullandıysanız siz de bu yazımda bahsettiğim şeylere biraz öyle bakmış olabilirsiniz.

Ancak meraklanmayın, burada sisteminize zarar verebilecek herhangi bir şey anlatmadım. Bizim için son derece yararlı olacak temel git komutlarını ele aldık.

Buradaki 5-6 git komutunu değişik klasörlerde tekrar edin, iyi anladığınıza emin olun. Çok yararını göreceğinize sizi temin ederim.
