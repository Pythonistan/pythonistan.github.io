Palindrome Programımız

9/Aug 2016
karakter dizileri stringler
Like Google +1 Tweet

Nihayet bir kaç gün önceki yazımda geçen Palindrome programımızı bugün ele alma şansımız olacak.

Tersten Okunduğunda Aynı Olan Metin Palindrom

Orjinal kodumuzu ve programımızın özetini tekrar hatırlayalım:

# -*- coding: utf-8 -*- 
def palindrome(s):
    return 0 if s == s[::-1] else -1

text = 'ey edip adanada pide ye'

print palindrome(text)

Yukarıda gördüğünüz program bir metnin palindrom olup olmadığını kontrol ediyor, eğer metin palindrome ise ekrana 0(pozitif), değilse -1(negatif) yazdırıyor.

Masaüstüne Kayıt

Bu programı Sublime kurulum makalemi okurken satır satır yazmıştınız, değil mi? :)) Sitemizi yeni takip etmeye başlayanlar buradaki kodları satır satır yazmayı unutmasınlar.

Arkadaşlar, kod ancak böyle öğreniliyor. O yüzden lütfen es geçmeyin.

Kodları yazdıktan sonra Masaüstüne yada kolay ulaşabildiğiniz bir konuma palindrom.py olarak kaydedin.

Prosedürler

Prosedürler konusuna Notepad ile ilk programımızı yazarken biraz değinmiştim. Prosedürler bir programı genel bir çerçeve haline getirmek için en elverişli araç.

Programımın sadece belli bir karakter dizisi için değil bütün karakter dizileri için geçerli olmasını istediğim için bir prosedür oluşturdum. Burada prosedürün parametresi olan (s) herhangi bir stringi ifade ediyor.

Test etmek istediğimiz herhangi bir karakter dizisini prosedürümüzle birlikte şu şekilde kullanabiliyoruz. print palindrome("karakter dizisi") yada print palindrome(metin) Burada metin karakter dizimizi tutan(içeren) bir değişkendir.

Tersi olsaydı yani sadece belli bir karakter dizisinin palindrome olup olmadığını anlamak için aşağıdaki gibi prosedür içermeyen bir program yazabilirdim:

# -*- coding: utf-8 -*- 
s = 'ey edip adanada pide ye'
print 0 if s == s[::-1] else -1

Aralarınızdan bazılarının “yahu bu daha kolay ve kısaymış” dediğinizi duyuyor gibiyim.

Arkadaşlar, bizim amacımız istediğimiz iş için genel bir çerçeve oluşturmak. Bu örnek kolay olduğu için prosedür kullanma yada kullanmama pek fark etmemiş olabilir. Ancak ileride çok daha karmaşık problemleri çözen kodlarda prosedür kullanmaktan başka çare kalmayacak.

Ayrıca yukarıdaki kodları birden fazla karakter dizisi için denemeye çalışın. Prosedür kullandığımızda metin1, metin2, metin3 gibi bir çok karakter dizisi oluşturup hepsini basitçe print palindrome(metin1) v.s. şeklinde test edip kullanabilirim. (Bkz. Aşağıda Test Örnekleri Ekran Görüntüleri)

Diğer seçenekte ise her seferinde s değerini silmek ve yeni değeri girmek zorunda kalacağım.

Gelelim Orjinal Kodlara

Prosedürümüzdeki kodlara yakından bakalım.

  • İlk satırda prosedürümüzün ismini belirliyor ve prosedürü tanımlıyoruz.
  • def kelimesi ingilizce define kelimesinin kısaltılmışıdır ve define tanımla demektir. Prosedür tanımlarken her zaman ilk olarak def kelimesi, sonra prosedürümüzün ismi ve daha sonra parantez, varsa içinde parametre(ler) ve en son iki nokta üst üstüste “:” gelir. Örnek: def prosedur_ismi(_parametreler_):
  • Parantez içine prosedürümüzün parametre(ler)si gelecek. Tüm prosedürler parametre almak zorunda mıdır? Hayır almak zorunda değil, ancak almasa bile prosedürün isminden sonra gelen parantezleri koymak zorundasınız. Aksi halde Python hata verir.
  • İkinci satırımızda hemen return ile prosedürümüze return‘den sonraki bölümü yürüt ve bunu return ile döndür diyoruz.
  • Prosedürler genelde hep bir şeyler döndürürler (return) ve biz döndürülen değeri görmek istiyorsak print ile ekrana yazdırma gerçekleştiririz.
  • Döndürülen değer ekrana yazdırılmayacak başka bir prosedürde/programda kullanılacaksa o halde diğer prosedürün parametresi olarak kullanılır.
  • Sıra geldi return‘den sonraki kısma. Burada artık programımızın işlevi açığa çıkıyor. Program karakter dizisini temsil eden değişkeni (s) kendisinin tersi ile (s[::-1]) karşılaştırıyor. Buradaki == operatörü “eşitlik” anlamındadır.
  • Herhangi bir karakter dizisini [::-1] ile kullanarak tersten okutabilirsiniz. Burada “-1” bir adım geri git anlamına gelmektedir.
  • Bu karşılaştırmanın kendisi bir boolean (True/False) değer ürettiği için karşılaştırmanın sonucunu hemen return fonksiyonuna döndürdük. Gelen değer eğer True (Doğru) ise ekrana 0, Yanlış (False) ise ekrana -1 yazdırılacak. If kelimesi “eğer … ise”, else “değil ise” anlamındadır. (Bkz. Aşağıdaki Test Örnekleri - Rakamlı Örnek)
  • Karşılaştırmanın sonucunda ekrana 0 ve 1 değerleri yazdırma yerine daha detaylı bir şey yazdırabiliriz. Kodumuzu şu hale çevirmemiz mümkün: return "Evet, bu bir palindrom." if s == s[::-1] else "Hayır, bu bir palindrom değil." (Bkz. Aşağıdaki Test Örnekleri - Yazılı Örnek)
  • Normalde eşitlik ifadesi if else bloğunu hiç kullanmadan karşılaştırma yapmaya olanak sağlıyor. Böyle bir kullanımı 3. varyasyonda göstereceğim.

Daha Anlaşılır ve Uzun If Else Bloklu 2. Varyasyonumuz

Şimdi aşağıda if else bloğunun daha açık yazıldığı bir varyasyon yazalım.

Bu kez kodumuz biraz uzayacak, ancak ziyan yok, bazen kodda açıklık kısalıktan evladır.

Yazacağımız kod bir prosedür içermesin. If else bloklarından hangisinin daha iyi olduğunun kararını size bırakıyorum. (Bkz. Aşağıda Solda Test Örnekleri - If Else Örneği)

cumle = 'ey edip adanada pide ye' # test cumlesi

palindrom_mu = 2  # gecici degisken ve başlangıç değeri
if cumle[0:] == cumle[::-1]:        # karşılaştırma yap
    palindrom_mu = 0                # aynıysa değ. değerini 0 yap
    print palindrom_mu              # değeri ekrana yaz
else:                           # karşılaştırma aynı değilse
    palindrom_mu = -1               # değişken değerini -1 yap
    print palindrom_mu              # değeri ekrana yaz

Burada ne mi yaptık? palindrom_mu adlı geçici bir değişken oluşturup ona bir ilk değer atadık. Bu ilk değer (2) önemli değil. Siz kendinize göre başka bir değer verebilirsiniz.

Daha sonra if karşılaştırma operatörü ile cumle değişkenine atanan karakter dizisi ile bu karakter dizisinin sondan okunuşunu karşılaştırdık.

Eğer bu karşılaştırmanın sonucu evet ise (yani True) o halde if‘den hemen sonra gelen blok yürütülecek. palindrom_mu değişkeninin değeri 0 olacak ve ekrana yazdırılacak.

Yok eğer yukarıda if bloğundaki karşılaştırma sonucu hayır ise (yani False) o halde else den sonra gelen blok yürütülecek. palindrom_mu değişkeninin değeri -1‘e eşitlenecek ve bu değer ekrana yazdırılacak.

palindrom_mu değişkenine başlangıçta (initial) atanan 2 değerinin programın hiç bir aşamasında çalışma fırsatı bul(a)madığına dikkat edin lütfen. O yüzden yukarıda önemli değil dedim.

Son olarak if ve else satırlarından sonra gelen satırlardaki tablara (indentation) dikkat edin lütfen! Tablama yapmazsanız hata alırsınız.

If Else Bloku İçermeyen 3. Varyasyonumuz

Yukarıdaki kodlarda gördüğünüz (==) operatör “eşit mi?” anlamını doğal olarak içinde taşır ve bu yüzden aslında if else bloğunu hiç kullanmadan karşılaştırma yapmaya imkan tanır.

Lütfen aşağıdaki kodlara bir göz atın ve yine lütfen kendiniz yazarak kodları deneyin.

# -*- coding: cp1254 -*-
def palindrom_mu(s):
    sonuc = (s==s[::-1]) - 1 # Boolean ifadeyi sayısal bir
    return sonuc             # değişken değere atama

metin1 = 'ey edip adanada pide ye'
metin2 = "mazdam"
metin3 = "madam"

print palindrom_mu(metin1)
print palindrom_mu(metin2)
print palindrom_mu(metin3)

Yukarıdaki çözüm oldukça önemli bir konsepti ifade ediyor. (Altta Sağda Boolean1 Değeri Sayısal Değere Dönüştürme Örneği Ekran Görüntüsü)

Normalde (s==s[::-1]) ifadesinin bir karşılaştırma olduğunu ve True(1) yada False(0) döndürdüğünü biliyoruz. Bu ifadeyi sayısal bir operatör ile beraber kullandığımızda (burada “- 1”) True değeri 1 ifadesi, False değeri 0 ifadesini alır.

İsterseniz yukarıdaki işlemde en sonraki “-1” ifadesini silip sonucu deneyin. (sonuc = (s==s[::-1]) olarak) Python interpreter size herhangi bir sayısal değer vermeden direkt True yada False çıktısını döndürecektir. (Üstte Solda Boolean Değeri Normal Kullanma Örneği Ekran Görüntüsü)

Dolayısıyla yapılan işlem şu: sonuc adlı değişkene atanacak değer karşılaştırma doğruysa 1 - 1 = 0, yanlışsa 0 - 1 = -1 olacak. Zaten bu da bizim almayı istediğiz sonuçtu. Yapmayı istediğimiz şeyi başardık.
